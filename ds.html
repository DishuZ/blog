# 数据结构

1. 时间复杂度

⌛️ 答： 时间复杂度 是执行算法所需的计算工作量，因为整个算法的执行时间与基本操作重复执行的次数成正比，所以将算法中基本操作的执行次数作为时间复杂度的度量 。一般情况下，按照基本操作执行次数最多的输入来计算时间复杂度，并且多数情况下我们把最深层循环内的语句所描述的操作作为基本操作。

3. 什么是二叉排序树？以及它的原理，算法。（二叉排序树的查找过程）

⌛️ 答：

定义： 二叉排序树又称二叉查找树，它或者是一颗空树，或者满足以下性质的二叉树：

  1. 若左子树不空，则左子树上所有结点的值均小于根节点的值；
  2. 若右子树不空，则右子树上所有结点的值均大于根节点的值；
  3. 左右子树也分别是二叉排序树。

从这三点性质中，我们可以发现二叉排序树的定义是递归的，并且树中的每一个结点都满足 左子树的值 < 根结点的值 < 右子树的值。

原理步骤：

  1. 若根结点的值等于查找的关键字，成功。
  2. 否则，若小于根结点的关键字值，递归查左子树。
  3. 若大于根结点的关键字值，递归查右子树。
  4. 若子树为空，查找不成功。
4. 哈夫曼树

⌛️ 答：

定义：

给定 n 个权值作为 n 个叶子结点，构造一棵二叉树，若带权路径长度达到最小，称这样的二叉树为最优二叉树，也称为哈夫曼树(Huffman tree)。

构造方法：

假设有 n 个权值，则构造出的哈夫曼树有 n 个叶子结点。n 个权值分别设为 w1w_1w1​、w2w_2w2​、……… 、wnw_nwn​，则哈夫曼树的构造规则为：

  1. 将 w1w_1w1​、w2w_2w2​、……… 、wnw_nwn​ 看成是有 n 棵树的森林 F（每棵树仅有一个结点）；
  2. 在森林中选出两个根结点的权值最小的树进行合并，作为一棵新树 T 的左、右子树，且新树 T 的根结点的权值为其左、右子树根结点权值之和；
  3. 从森林 F 中删除选取的两棵树，并将新树 T 加入森林 F；
  4. 重复 2)、3) 步，直到森林中只剩一棵树为止，该树即为所求得的哈夫曼树。

特点：

  1. 权值越大的结点，距离根节点越近；
  2. 树中没有度为一的结点。

应用： 哈夫曼编码，减少编码的长度。哈夫曼编码就是长度最短的前缀编码。

5. 简要介绍一下最小生成树

⌛️ 答：

要了解最小生成树，可以先了解生成树的概念。

无向图G中的生成树是具有图G中所有顶点，但边数最少的连通子图。无向图可以有多个生成树。

带权图的生成树中，总权值最小的生成树称为最小生成树，或者说，最小生成树的拓扑图中的路径长度之和最小。

构建最小生成树，可以使用 Prim 普里姆算法和 Kruskal 克鲁斯卡尔算法。

Prim 的基本思想是：首先，选取图中的任一顶点作为一棵树，从与这棵树相接的边中选取权值最小的边，然后将该边及连接的顶点并入到树中，此时得到有两个顶点的树。接着，从与这棵树相接的边中选取权值最小的边，然后将该边及连接的定点吗并入到树中，此时得到有三个顶点的树。以此类推，直到图中所有的顶点都并入到树中为止，得到的就是最小生成树。

Kruskal 的基本思想是：最开始，将图中所有的边按照权值从小到大排序，然后从小到大遍历这些边。首先，选取权值最小的边及连接的顶点并入树中；接着，选取权值第二小的边，若该边并入树中不构成环，就并入树中；接着选取权值第三小的边，以此类推，直到图中所有的顶点都并入树中为止，得到的就是最小生成树。

最小生成树有它的实际应用，如 网线部署-如何用最小的代价/最少的网线让全村能够上网。

6. 简要介绍一下最短路径

⌛️ 答：

最短路径可以分为两类：单源最短路径和多源汇最短路径，我们熟知的 Dijkstra 迪杰斯特拉算法和 Flyod 佛洛依德算法就分别属于这两类。

Dijkstra 是构建图中一个点到其余顶点的最短路径，它的算法思想是：

首先分为集合 S 和集合 T，集合 S 存放已经并入的顶点，集合 T 存放暂未并入的顶点。

初始时，集合 S 中只有源点 V0V_0V0​，接着，不断地从集合 T 中选取到源点 V0V_0V0​ 的路径长度最小的顶点 VuV_uVu​，将 VuV_uVu​ 并入到集合 S 中，并以 VuV_uVu​ 为中转站，更新集合 T 中所有顶点到源点 V0V_0V0​ 的最短路径长度，直到集合 T 中的所有顶点都并入到集合 S 中为止，此时得到最短路径树。

Dijkstra 的时间复杂度是 O(n2)O(n^2)O(n2) ，但如果通过堆优化，时间复杂度可以达到 O(m∗log2(n))O(m*log_2(n))O(m∗log2​(n))。Flyod 的时间复杂度是 O(n3)O(n^3)O(n3) 。

Dijkstra 可以解决正权边的最短路径问题，但如果图中有负权边，那决定不能使用 Dijkstra。需要使用 Bellman-Ford 算法 O(m∗n)O(m*n)O(m∗n) 和 SPFA 算法 O(m)O(m)O(m) 来解决。

Flyod 则可以用来解决有负权边、无负环的问题。

并发一致性问题

⌛️ 答：

原因：破坏了事务的隔离性

问题：丢失数据、读脏数据、不可重复读（读时修改）、幻影读（读时插入）

解决方法：并发控制，通过封锁（获取锁、释放锁、检查锁状态）来实现，需要用户自己控制。而DBMS提供了事物的隔离级别，让用户以更轻松的方式处理并发一致性问题

# 软件工程

什么是软件工程？目前有哪几种主要的软件工程方法？

⌛️ 答： 概括地说，软件工程是指导计算机软件开发和维护的一门工程学科，用工程的概念、原理、技术和方法来开发和维护软件，结合经过时间验证的管理技术和目前能得到的最新的技术方法来开发出高质量的软件并进行有效的维护，这就是软件工程。它涉及程序设计语言、数据库、软件开发工具、系统平台等等方面的知识。
